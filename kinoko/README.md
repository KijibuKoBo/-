# 長岡きのこ採集記 🍄

新潟県・長岡市の山を **約20年** 歩いて記録した、石原 巖（いしはら いわお）の
きのこ採集アーカイブです。

> **アプリで記録 → ホームページで公開 → 書籍化** までを一本でつなぐプロジェクト。

採集したきのこを「おしゃれで格好いい図鑑」としてブラウザで管理し、
そのままWebサイトとして公開でき、最終的には印刷・書籍化できるデータ構造で
記録を残します。

---

## ✨ いまできること（フェーズ1：アプリ）

デザインカンプに合わせた、編集デザイン（縦書きヒーロー × 明朝体 × 森のダーク＆生成り）の
ホームページです。

**トップページ `index.html`**
- 🌲 **ヒーロー**：「長岡の森が育む、小さな命の記録。」
- 👤 **ABOUT**：石原 巖 の紹介
- 🍄 **GALLERY**：最近の採集記録をカードで抜粋（→「すべての記録を見る」で図鑑へ）
- 📝 **COLUMN**：採集のコツ・季節・毒きのこの見分け方などのコラム
- 📱 PC は左サイドナビ、スマホはハンバーガーメニューに自動で切替

**図鑑ページ `zukan.html`**
- 🔍 **検索・絞り込み**：和名／学名／場所／環境で検索、食毒（食用・毒・食不適）で絞り込み
- ↕️ **並び替え**：採集日・五十音・おすすめ度
- 📖 **詳細表示**：採集日／場所／標高／発生環境／共生樹／味・食べ方／覚え書き／似た毒きのこへの注意
- 📚 **図鑑モード**：1種1ページのレイアウトに切替（→ そのまま印刷・PDF化＝書籍化の下地）

データは [`web/data/records.json`](web/data/records.json)（きのこ）と
[`web/data/columns.json`](web/data/columns.json)（コラム）に入っています。
いまはサンプル（ナメコ・マイタケ・ムキタケ 等）なので、
**石原巖さんの実際のノート・写真に差し替え**ていきます。
写真は [`web/data/photos/`](web/data/photos/) に置くと自動で反映されます。

## 🚀 ローカルで見る

```bash
cd kinoko/web
python3 -m http.server 8000
# → ブラウザで http://localhost:8000 を開く
```

> ※ `index.html` をダブルクリックで直接開くと、ブラウザの制限で
> `records.json` を読めないことがあります。上の簡易サーバー経由が確実です。

## 🗺 ロードマップ

| フェーズ | 内容 | 状態 |
| --- | --- | --- |
| **1. アプリ** | 採集記録を入力・閲覧する図鑑アプリ | 🟢 土台完成（このリポジトリ） |
| **2. データ移行** | 石原巖さんのノート・写真を `records.json` に取り込み | ⚪️ これから |
| **3. ホームページ公開** | GitHub Pages で一般公開 | ⚪️ これから |
| **4. 書籍化** | 図鑑モードを版下に整え、印刷・製本 | ⚪️ これから |

詳細は [`docs/roadmap.md`](docs/roadmap.md) を参照。

## 📁 構成

```
kinoko/
├── README.md
├── docs/
│   ├── roadmap.md         … アプリ→HP→書籍化の進め方
│   └── data-model.md      … records.json の項目仕様（記録の付け方）
└── web/                   … フロントエンド（GitHub Pages 公開対象）
    ├── index.html         … トップページ（ヒーロー/ABOUT/GALLERY/COLUMN）
    ├── zukan.html         … 図鑑ページ（全記録・検索・絞り込み・図鑑モード）
    ├── style.css
    ├── app.js
    └── data/
        ├── records.json   … きのこ採集記録（ここを育てていく）
        ├── columns.json   … コラム
        └── photos/        … 写真を置く場所（置けば自動反映）
```
