# 管理ページの使い方（新しいキノコをHPに公開する）

`web/admin.html` を開くと、記録の追加・編集ができます。
「公開」ボタンを押すと、GitHub に保存され、数分でホームページに反映されます。

> 管理ページの住所（例）：`https://（公開URL）/admin.html`
> このページは検索エンジンに載らない設定（noindex）にしてあります。URLは身内だけで共有してください。

---

## 最初に1回だけ：GitHubトークンの登録

「公開」ボタンが GitHub に書き込むための鍵（トークン）を、1度だけ登録します。
トークンは **この端末のブラウザの中だけ** に保存され、外には送られません。

### 1. トークンを発行する
1. パソコン/スマホで GitHub にログイン
2. [Fine-grained トークン発行ページ](https://github.com/settings/tokens?type=beta) を開く
3. 「Generate new token」
   - **Token name**：`kinoko-admin` など分かる名前
   - **Expiration**：好きな期限（長め推奨）
   - **Repository access**：「Only select repositories」→ `KijibuKoBo/-` を選択
   - **Permissions** → Repository permissions の **Contents** を **Read and write** に
4. 「Generate token」→ 表示された `github_pat_…` をコピー（**この画面でしか見られません**）

### 2. 管理ページに登録する
1. `admin.html` を開く → 右上の **⚙ 設定**
2. **GitHub トークン** にコピーした文字列を貼り付け
3. オーナー＝`KijibuKoBo` / リポジトリ＝`-` / ブランチ＝公開しているブランチ名
   （GitHub Pages の公開ブランチに合わせる。通常 `main`）
4. **接続テスト** → 「✓ 接続OK」が出ればOK
5. **設定を保存**

> **合言葉（任意）**：設定すると、次回から管理ページを開く時に合言葉を聞かれます
> （その端末を他人に触られた時の簡易ロック）。本当の鍵はトークンなので、
> トークン自体を他人に渡さないことが一番大切です。

---

## キノコを追加する

1. 「編集する記録」で **＋ 新しいキノコを追加** を選択（初期状態）
2. **和名・採集日・食毒** は必須。あとは分かる範囲でOK
3. **写真**：スマホなら「写真」欄からその場でカメラ撮影できます
   （アップロード時に自動で軽く圧縮されます）
4. **プレビュー** で見え方を確認
5. **🍄 公開する** を押す → 「✅ 公開しました」が出れば完了
6. 1〜2分ほどでホームページに反映されます

## 既存のキノコを直す

1. 「編集する記録」から直したいキノコを選ぶ → フォームに内容が入ります
2. 修正して **🍄 公開する**
3. 写真を選び直せば差し替え、選ばなければ元の写真のまま

---

## うまくいかない時

| 症状 | 対処 |
| --- | --- |
| 「先にトークンを登録して」 | ⚙設定でトークンを保存 |
| 接続テストが ✗ (404) | ブランチ名・リポジトリ名が公開先と合っているか確認 |
| 接続テストが ✗ (401/403) | トークンの期限切れ、または Contents 権限が無い |
| 公開したのにHPが変わらない | 反映に1〜2分。スマホは再読み込み（キャッシュ更新） |

## 仕組み（参考）

- 「公開」＝ `kinoko/web/data/records.json` と写真 `kinoko/web/data/photos/`
  を GitHub にコミットしているだけ
- GitHub Pages が自動で作り直すので、サーバーは不要
- 写真もデータも全部リポジトリに残る＝そのまま書籍化の素材になる
