# 記録データの仕様（records.json の付け方）

採集記録は `web/data/records.json` の `records` 配列に、1きのこ＝1オブジェクトで書きます。
ここでは各項目の意味と書き方をまとめます。**必須は数項目だけ**で、分からない欄は空 `""` や
省略でOK。後から少しずつ埋められます。

## ファイル全体の形

```json
{
  "meta": {
    "title": "長岡きのこ採集記",
    "author": "石原 巖",
    "areaLabel": "新潟県 長岡市 周辺の山",
    "yearsLabel": "約20年の記録"
  },
  "records": [
    { /* ↓ 1件のきのこ */ }
  ]
}
```

## 1件（きのこ）の項目

| キー | 意味 | 例 | 必須 |
| --- | --- | --- | :---: |
| `id` | 重複しない管理番号 | `"k016"` | ✅ |
| `wamei` | 和名 | `"ナメコ"` | ✅ |
| `kana` | よみがな | `"なめこ"` | |
| `gakumei` | 学名（斜体で表示） | `"Pholiota microspora"` | |
| `family` | 科名 | `"モエギタケ科"` | |
| `aka` | 別名・地方名（配列） | `["ナメラッコ"]` | |
| `edibility` | 食毒区分 | `"食用"` / `"毒"` / `"食不適"` | ✅ |
| `date` | 採集日（YYYY-MM-DD） | `"2024-10-28"` | ✅ |
| `area` | 採集地（ざっくり可。非公開なら `"（非公開）"`） | `"東山 ブナ帯"` | |
| `elevation` | 標高（数値・m） | `540` | |
| `habitat` | 発生環境 | `"ブナの倒木・立ち枯れ"` | |
| `host` | 共生樹・宿主 | `"ブナ"` | |
| `season` | 季節タグ | `"秋"` / `"晩秋"` / `"冬"` | |
| `weather` | 天候・気温 | `"雨上がり・気温12℃"` | |
| `quantity` | 採集量 | `"ザル一杯"` | |
| `taste` | 味・食べ方 | `"味噌汁・なめこおろし"` | |
| `notes` | 覚え書き・エピソード | `"毎年同じ倒木に出る"` | |
| `caution` | 注意（似た毒きのこ等） | `"ツキヨタケに注意"` | |
| `lookalikes` | 似たきのこ（配列） | `["ツキヨタケ"]` | |
| `rating` | おすすめ度（0〜5） | `5` | |
| `photo` | 写真パス（空なら自動でプレースホルダ） | `"data/photos/nameko.jpg"` | |

### 食毒区分 `edibility` の値
- `"食用"` … 食べられる（緑バッジ）
- `"毒"` … 毒きのこ（赤バッジ）
- `"食不適"` … 無毒だが食用に向かない（灰バッジ）

※ 食べられるが猛毒と紛らわしい種は `"食用"` にしたうえで、
`caution` と `lookalikes` に必ず警告を書いてください（アプリが「⚠ 似た毒きのこに注意」を表示します）。

## 書き方の例（最小）

```json
{
  "id": "k016",
  "wamei": "ブナハリタケ",
  "edibility": "食用",
  "date": "2023-10-15"
}
```

## 書き方の例（しっかり）

```json
{
  "id": "k017",
  "wamei": "ブナハリタケ",
  "kana": "ぶなはりたけ",
  "gakumei": "Mycoleptodonoides aitchisonii",
  "family": "ブナハリタケ科",
  "aka": ["カノカ", "スギカノカ"],
  "edibility": "食用",
  "date": "2023-10-15",
  "area": "東山 ブナ帯",
  "elevation": 600,
  "habitat": "ブナの倒木・枯れ木に重なり合って群生",
  "host": "ブナ",
  "season": "秋",
  "weather": "晴れ",
  "quantity": "大株",
  "taste": "天ぷら・炒め物。芳香あり。古いと苦味。",
  "notes": "裏面が針状。良い香りがする。",
  "caution": "",
  "lookalikes": [],
  "rating": 4,
  "photo": "data/photos/bunaharitake-2023.jpg"
}
```

## 写真の置き場所

- `web/data/photos/` フォルダを作り、そこに画像を置く
- `photo` には `"data/photos/ファイル名"` と書く
- 横向き・長辺2000px以上を推奨（書籍化で活きる）
- 写真が無い間は `photo` を空 `""` にしておけば、図鑑風のプレースホルダが出ます

## 入力時のコツ

- まず `id` `wamei` `edibility` `date` の4つだけでも登録してOK
- 写真と覚え書き（`notes`）は後から足すと、ぐっと「本らしく」なります
- JSONはカンマ・引用符が崩れると全体が読めなくなるので、
  1件足したら一度ブラウザで開いて表示を確認すると安全です
