# ロードマップ：アプリ → ホームページ → 書籍化

石原 巖さんの約20年分のきのこ採集記録を、デジタルで整理し、公開し、
最終的に一冊の本にまとめるための進め方です。

---

## フェーズ1 ── 図鑑アプリ（土台）🟢 完成

採集記録を「おしゃれな図鑑」として閲覧できるWebアプリ。

- ギャラリー／検索／絞り込み／詳細／図鑑モード
- データは `web/data/records.json` の1ファイルで完結（サーバー不要）
- どの端末でも同じURLで閲覧

**この時点の成果物**：`kinoko/web/` 一式。

---

## フェーズ2 ── データ移行（記録を育てる）⚪️

石原巖さんの実際の採集ノート・写真を `records.json` に取り込む段階。

進め方の案：

1. **記録の棚卸し**
   - 手帳・カレンダー・写真フォルダから「いつ・どこで・何を」を拾い出す
   - 写真のExif（撮影日・GPS）から採集日・場所を補完できる
2. **入力**
   - まずは1件ずつ `records.json` に追記（項目は [`data-model.md`](data-model.md) 参照）
   - 写真は `web/data/photos/` に置き、`"photo": "data/photos/nameko-2024.jpg"` のように指定
3. **（任意）入力フォーム化**
   - 件数が増えたら、ブラウザ上で入力できる管理画面を追加
   - 既存のカワサキ発注書アプリと同様、Google スプレッドシート／Apps Script を
     バックエンドにする案もあり（家族で分担入力しやすい）

**目標**：実データ 30〜50件で「図鑑として読み応えのある」状態に。

### 写真について
- 図鑑・書籍化を見据え、**1種につき横向き・高解像度（長辺2000px以上）** を1枚は確保
- 発生環境（木・地面）が分かる引きの写真もあると本で映える

---

## フェーズ3 ── ホームページ公開 ⚪️

GitHub Pages で一般公開する。

1. GitHubリポジトリの **Settings → Pages** で公開ブランチ／フォルダを指定
   - 公開対象：`kinoko/web/`（または `web/` をルートにする構成へ調整）
2. 公開URLを家族・知人に共有
3. （任意）独自ドメイン、OGP画像（SNSシェア時のサムネ）、アクセス解析

**ポイント**：このアプリは静的ファイルだけで動くので、Pagesにそのまま載る。

---

## フェーズ4 ── 書籍化 ⚪️

「図鑑モード」を版下（レイアウト原稿）に育て、印刷・製本する。

- **章立て案**：季節別（春・梅雨・夏・秋・晩秋〜冬）／環境別（ブナ林・アカマツ林・カラマツ林）／食毒別
- **1種1ページ**：大きな写真＋和名・学名・科＋採集データ＋覚え書き
- **巻頭・巻末**：長岡の山ときのこ歩き20年のエッセイ、似た毒きのこ早見表、採集マナー
- **出力方法の候補**
  1. 図鑑モード／印刷スタイルから **ブラウザでPDF化** → ネット印刷・製本（手軽）
  2. PDFを元に InDesign 等で本格レイアウト（こだわる場合）
  3. 写真集印刷サービス、または自費出版・同人誌印刷所へ入稿

**目標**：家族・仲間に配れる私家版 → ご当地（長岡）の自然本として一般流通も視野に。

---

## 次の一手（おすすめ）

1. このアプリをブラウザで開いて、サンプルの見え方を確認
2. 石原巖さんの「いちばん思い出に残る1種」から実データ1件を入力してみる
3. 写真を1枚入れて、図鑑モードでの仕上がりを見る

ここまで回れば、あとは件数を増やしていくだけで本に近づきます。
